## 🔑 Getting Your Railway Token

### Step 1: Create Token
1. Visit: **https://railway.app/account/tokens**
2. Click **"Create Token"**
3. Give it a name: `quant-deployment`
4. Copy the token (starts with `railway_token_...`)

### Step 2: Set Environment Variable
```bash
# For current session
export RAILWAY_TOKEN=railway_token_xxxxxxxxxxxxxxxxxxxxx

# Or add to ~/.bashrc or ~/.zshrc for persistence
echo 'export RAILWAY_TOKEN=railway_token_xxxxxxxxxxxxxxxxxxxxx' >> ~/.bashrc
source ~/.bashrc
```

### Step 3: Verify
```bash
railway whoami
# Should show your Railway username
```

## 🚀 What the Scripts Do

Both scripts perform these steps automatically:

1. ✅ **Check Authentication** - Verify Railway token
2. ✅ **Install CLI** - Install Railway CLI if needed
3. ✅ **Initialize Project** - Create Railway project `quant-backend`
4. ✅ **Set Variables** - Configure environment variables:
   - `PROJECT_NAME=QuantBacktesting`
   - `VERSION=1.0.0`
   - `ENVIRONMENT=production`
   - `DATABASE_URL=sqlite+aiosqlite:///./quant.db`
   - `SECRET_KEY=<auto-generated>`
   - `JWT_SECRET_KEY=<auto-generated>`
5. ✅ **Deploy** - Upload code to Railway
6. ✅ **Get URL** - Fetch your live deployment URL
7. ✅ **Test** - Provide test commands

**Total Time**: 3-5 minutes

## 🐛 Troubleshooting

### "Unauthorized. Please login"
**Problem**: No Railway token set  
**Fix**:
```bash
export RAILWAY_TOKEN=your_token_here
railway whoami  # Verify
```

### "Failed to initialize Railway project"
**Problem**: Project name conflict  
**Fix**: Edit script and change `PROJECT_NAME="quant-backend-2"`

### "Deployment failed"
**Problem**: Build error  
**Fix**:
```bash
railway logs  # Check error logs
```

Common causes:
- Missing dependencies → Check `requirements.txt`
- Import errors → Already fixed in latest code
- Environment variables → Check with `railway variables`

### Build succeeds but app won't start
**Problem**: Port binding  
**Fix**: Railway sets `$PORT` automatically, our start command uses it:
```bash
uvicorn app.main:app --host 0.0.0.0 --port $PORT
```
